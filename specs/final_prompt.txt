# Project Specification

## User Requirement
build a backend for online bakery shop

## Selected Technology Stack

### 1. Web Development

### 2. Backend
**Why chosen:** The Backend is required to manage and process data related to the online bakery shop such as orders, inventory, user authentication etc.
**Purpose:** This option enables managing and processing of data for the backend functionality

### 3. REST
**Why chosen:** REST (Representational State Transfer) is a well-established and widely-used API design pattern, making it easier for developers to understand and implement the backend for an online bakery shop.
**Purpose:** Enables easy communication between the frontend and backend using standard HTTP methods such as GET, POST, PUT, DELETE.

### 4. JavaScript/TypeScript
**Why chosen:** For simplicity, quick development and easy integration with frontend (if using React or Angular), JavaScript/TypeScript is chosen.
**Purpose:** Enables efficient server-side logic implementation for the online bakery shop.

### 5. NestJS
**Why chosen:** NestJS is built on top of Express and provides additional features like modularity, DI (Dependency Injection), and TypeScript support making it suitable for large-scale projects.
**Purpose:** Enables the development of a scalable backend for an online bakery shop.

### 6. NGINX
**Why chosen:** Popular choice for handling high traffic applications and reverse proxying in Node.js projects
**Purpose:** Enables efficient routing and management of requests to the NestJS application

## Technology Stack Summary
Web Development → Backend → REST → JavaScript/TypeScript → NestJS → NGINX
