# Project Specification

## User Requirement
build a backend for online bakery shop that sells cakes

## Selected Technology Stack

### 1. Web Development

### 2. Backend
**Why chosen:** As the user requirement is to build a backend for an online bakery shop that sells cakes, it is necessary to focus on the backend development.
**Purpose:** This option enables the creation of database models, APIs, and server-side logic essential for managing inventory, processing orders, and facilitating interactions between the frontend application and the database.

### 3. REST
**Why chosen:** REST is widely adopted and supported by most programming languages, making it easier to find resources for development.
**Purpose:** Allows creation of stateless services that can be easily scaled horizontally.

### 4. Java
**Why chosen:** Java is widely used for large-scale, enterprise applications and has robust libraries for building backends.
**Purpose:** Allows for efficient development of a secure and scalable backend solution.

### 5. Spring Boot
**Why chosen:** Popularity, large community support, and extensive documentation make Spring Boot an ideal choice for a backend application.
**Purpose:** Enables rapid development of a robust Java-based RESTful API.

### 6. Tomcat
**Why chosen:** Tomcat is a popular and well-documented servlet container for Spring Boot applications.
**Purpose:** It enables easy deployment and running of the online bakery shop backend

## Technology Stack Summary
Web Development → Backend → REST → Java → Spring Boot → Tomcat
